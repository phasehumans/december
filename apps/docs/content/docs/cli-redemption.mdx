December CLI includes a secure, offline-compatible credit redemption system. This mechanism allows developers to redeem promotion and credit codes directly via the command line.

## Architecture

The system utilizes cryptographic hashing to verify transactions securely without exposing raw promotional secret strings in transit:

    chart={`graph TD
    A[december redeem CODE] --> B[Generate SHA-256 Hash]
    B --> C[Post to Server Endpoint]
    C --> D[Rate Limiter Check]
    D -->|Under Limits| E[Query Hashed Table]
    E -->|Valid Match| F[Credit Transaction & Claimed Flag]
    E -->|Invalid Match| G[Reject Action]`}
/>

## Code Specifications

- **SHA-256 Hashing**: Codes are hashed client-side before submission to the API to minimize interception risks.
- **Transaction-Bound**: Promotional tokens are bound to a single user account context. Once a code is validated, it is flagged as claimed and locked from further redemption.
- **Limiting & Security**: Rate limiters throttle verification endpoints to prevent brute-force generation attacks.

## Redeeming via CLI

To claim promotional credits, execute the `redeem` command in your terminal:

```bash
december redeem PROMO-SUMMER-2026
```

If the code is valid, your credit balance will immediately show the updated amount.
